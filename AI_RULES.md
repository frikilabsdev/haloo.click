# AI_RULES

Reglas obligatorias de trabajo para el agente en este proyecto.

Antes de modificar cualquier archivo, siempre debo:

1. Revisar `PROJECT_CONTEXT.md`.
2. Analizar dependencias del cambio (módulos, rutas, DB, servicios y configuración).
3. Explicar el impacto del cambio antes de implementarlo.
4. No romper APIs existentes ni contratos de datos actuales.
5. No duplicar lógica; reutilizar funciones/módulos existentes cuando aplique.
